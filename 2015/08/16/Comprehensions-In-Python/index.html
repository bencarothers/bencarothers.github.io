Comprehensions in Python

A Baby Step Toward Pythonic Code

Posted by Ben Carothers on August 16, 2015

Python is a really elegantly constructed language that comes packaged with a large library of natural, powerful features. I see the language constantly bombarded with criticisms that largely seem to be coming from developers that fail to adopt a pythonic style of code.

The for loop is one of the first learned and most used snippets of code for any new language. Python is no exception to this fact 

numbers = [1,2,3]
numbers_squared = []
for number in numbers:
    number **= 2
    numbers_squared.append(number)

Here we have a list of 3 numbers. All we're looking to do is take each and square it, so that numbers becomes a list of each of its elements squared. So what's wrong with this code? Functionally, nothing. We end up with numbers_squared and can carry on with whatever task we we're initially trying to tackle. But is this idiomatic, pythonic code? Not quite.

Comprehensions

List comprehensions are a super easy way to make your code much more readable, cleaner and more efficient. They're simple and powerful and, personally, I think they should be taught right along with for loops.

So, how could we accomplish the same functionality using comprehensions?

numbers = [1,2,3]
numbers_squared = [number**2 for number in numbers]

Four lines are quickly condensed to one and the code is both more beautiful and also more readable.

Similarly we could do a few more involved operations

numbers = [1,2,3]
numbers_exponentiated = [[number**2,number**3,number**4] for number in numbers]

Comprehensions also can be nested or used to construct dictionaries. Here we're creating a dictionary with the numbers as keys and squares as values: 

numbers = [1,2,3]
numbers_squared = {number:number**2 for number in numbers}

Python is really well constructed and full of features so don't knock it too hard before you do a little bit of exploring.
